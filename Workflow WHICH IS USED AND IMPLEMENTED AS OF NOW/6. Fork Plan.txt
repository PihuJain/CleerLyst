## PHASE 6 — DATASET INGESTION PIPELINE (CORE LOGIC)

### New API Route

```
POST /api/admin/datasets/upload
```

Backend does:

1. Accept file
2. Parse server-side
3. Admin maps:

   * identifier column
   * visible fields
4. For each row:

   * hash identifier
   * encrypt payload
   * store as independent record
5. Destroy raw file (or encrypt + archive)

Frontend **never** touches rows.

## PHASE 7 — STUDENT SIDE (READ-ONLY BY DESIGN)

### Student Routes

```
src/app/
├── feed/page.tsx
├── dataset/[id]/page.tsx
```

### Feed API

```
GET /api/me/feed
```

Returns:

* dataset title
* type
* timestamp
* status (available / action required)

### Record API

```
GET /api/datasets/{id}/me
```

Backend flow:

1. Identify logged-in user
2. Fetch hashed identifiers
3. Match against dataset_records
4. Decrypt payload
5. Return ONLY allowed fields

If no match:

```json
{ "matched": false }
```

**Same response shape. No inference leaks.**
